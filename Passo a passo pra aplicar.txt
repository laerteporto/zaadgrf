Passo a passo pra aplicar:

Copie ad_monitor_write.ps1 pra C:\Scripts\ no WinServer2022
Crie a Scheduled Task (comando acima ou via UI)
Rode a task uma vez manualmente e confirme que C:\Scripts\ad_monitor_output.json tem conteúdo
Atualize o conteúdo do ad_monitor.conf do agente pra nova linha de UserParameter
Restart-Service "Zabbix Agent"
Teste local: zabbix_agentd.exe -t ad.monitor.data — deve responder na hora, sem demora
Execute now no item ad.monitor.data no Zabbix, e confere o zabbix_agentd.log


Caminho mais confiável: cria pela interface do Task Scheduler (menos chance de erro de sintaxe):

Abra Task Scheduler (taskschd.msc)
Action → Create Task (não "Create Basic Task", pra ter mais controle)
Aba General:


Name: AD Monitor Collector
Marque "Run only when user is logged on"
Marque "Run with highest privileges"
Em "Configure for": Windows Server 2019/2022

Aba Triggers → New:
Begin the task: On a schedule
Marque "One time"
Marque "Repeat task every" → 5 minutes → for a duration of → Indefinitely

Aba Actions → New:
Action: Start a program
Program/script: powershell.exe
Add arguments: -NoProfile -ExecutionPolicy Bypass -File "C:\Scripts\ad_monitor_write.ps1"

Aba Settings 
Marque - Allow task to be run on demand
Marque - Run task as soon as possible after a scheduled start is missed
Marque - if the task fails, restart every - 1 minute - 3 times
Marque - if the running task does not end when requested, force it top stop

Do not start a new instance
